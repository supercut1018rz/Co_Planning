QUICK DEPLOYMENT GUIDE - WINDOWS
===============================================

⚡ Deploy in 5-10 minutes on Windows!

===============================================
STEP 1: INSTALL DOCKER DESKTOP (Windows)
===============================================

A. Download Docker Desktop
   
   1. Visit: https://www.docker.com/products/docker-desktop
   2. Click "Download for Windows"
   3. Run the installer (Docker Desktop Installer.exe)

B. Enable WSL 2 (Recommended)
   
   Open PowerShell as Administrator:
   
   wsl --install
   
   Restart your computer when prompted.

C. Start Docker Desktop
   
   1. Launch Docker Desktop from Start Menu
   2. Wait for Docker to start (whale icon in system tray)
   3. Accept the terms if prompted

D. Verify Installation
   
   Open PowerShell or Command Prompt:
   
   docker --version
   docker compose version
   
   Expected output:
   Docker version 24.x.x
   Docker Compose version v2.x.x

===============================================
STEP 2: EXTRACT & CONFIGURE
===============================================

A. Extract Archive
   
   Method 1: Using File Explorer
   - Right-click howard-sidewalk-deployment-*.tar.gz
   - Select "Extract All..."
   - Choose destination folder
   
   Method 2: Using PowerShell
   - Open PowerShell
   - Navigate to download folder:
     cd Downloads
   - Extract (if you have tar):
     tar -xzf howard-sidewalk-deployment-*.tar.gz
   
   Method 3: Using 7-Zip
   - Right-click the file
   - 7-Zip → Extract Here

B. Open Folder
   
   cd howard-sidewalk-deployment-YYYYMMDD-HHMMSS\my-app

C. Create Environment File
   
   Copy template:
   copy .env.example .env
   
   Edit with Notepad:
   notepad .env
   
   Fill in these values:
   
   OPENAI_API_KEY=sk-proj-your-actual-key-here
   MAPILLARY_CLIENT_TOKEN=your-actual-token-here
   DATABASE_PASSWORD=ChangeMe_StrongPW
   
   Get keys:
   - OpenAI: https://platform.openai.com/api-keys
   - Mapillary: https://www.mapillary.com/dashboard/developers
   
   Save and close Notepad.

===============================================
STEP 3: DEPLOY
===============================================

A. Open PowerShell or Command Prompt
   
   Navigate to project folder:
   cd path\to\howard-sidewalk-deployment-*\my-app

B. Start Services
   
   docker compose up -d --build
   
   Wait 3-5 minutes for first build.

C. Monitor Progress
   
   Check status:
   docker compose ps
   
   View logs:
   docker compose logs -f
   
   Press Ctrl+C to exit logs.

===============================================
TROUBLESHOOTING
===============================================

Issue: Docker Desktop won't start
Solution:
   - Enable Virtualization in BIOS
   - Enable WSL 2: wsl --install
   - Restart computer
   - Reinstall Docker Desktop

Issue: "docker" command not found
Solution:
   - Make sure Docker Desktop is running (whale icon)
   - Restart PowerShell/Command Prompt
   - Check Docker Desktop settings

Issue: WSL 2 installation failed
Solution:
   Open PowerShell as Administrator:
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   Restart computer

Issue: Cannot extract .tar.gz
Solution:
   - Install 7-Zip: https://www.7-zip.org/
   - Or use built-in tar (Windows 10+):
     tar -xzf filename.tar.gz

Issue: OpenAI 401 error
Solution:
   - Check API key in .env file
   - Make sure no extra spaces
   - Verify OPENAI_BASE_URL is correct
   - Restart: docker compose restart

Issue: Port 3000 already in use
Solution:
   Find process using port:
   netstat -ano | findstr :3000
   
   Kill process (replace PID):
   taskkill /PID [PID] /F

Issue: Cannot access from browser
Solution:
   - Check Docker Desktop is running
   - Check firewall settings
   - Try: http://127.0.0.1:3000
   - Check containers: docker compose ps

Issue: Slow performance
Solution:
   - Allocate more resources in Docker Desktop
   - Settings → Resources → Advanced
   - Increase CPU and Memory
   - Recommended: 4 CPU, 8GB RAM

===============================================
WINDOWS-SPECIFIC TIPS
===============================================

1. Use PowerShell (recommended) or Command Prompt
   - PowerShell: More powerful, modern
   - CMD: Traditional, simple

2. File Paths
   - Use backslashes: C:\Users\YourName\Desktop
   - Or forward slashes work too: C:/Users/YourName/Desktop

3. Line Endings
   - .env file should use Unix line endings (LF)
   - Most editors handle this automatically

4. Performance
   - Store project files in WSL 2 for better performance
   - Or keep in Windows filesystem if needed

5. Firewall
   - Docker Desktop handles most networking
   - May need to allow Docker in Windows Firewall

===============================================
ALTERNATIVE: USING WSL 2 (Advanced)
===============================================

For better performance, run inside WSL 2:

1. Open WSL 2 (Ubuntu):
   wsl
   
2. Navigate to Windows folder:
   cd /mnt/c/Users/YourName/Downloads/my-app
   
3. Follow Linux instructions:
   sudo apt update
   sudo apt install docker.io docker-compose-plugin -y
   docker compose up -d --build
